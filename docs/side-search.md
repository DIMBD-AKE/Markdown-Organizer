문서 검색 사이드바 기능 명세

1. 개요

문서 검색 사이드바는 개발 중인 프로그램 내에서 프로젝트 문서를 빠르게 찾고 확인하기 위한 기능이다. 사용자는 검색 범위를 선택한 뒤, 마크다운 문서만 대상으로 문자열 또는 정규식 검색을 수행할 수 있다.

검색 결과에서 문서를 클릭하면 우측 문서 뷰어에 해당 문서의 전체 내용이 표시되며, 검색어가 발견된 위치로 바로 이동하고 검색된 부분을 확인할 수 있다.

⸻

2. 주요 기능

2.1 검색 범위 선택

사용자는 검색 대상 범위를 다음 중 하나로 선택할 수 있다.

옵션	설명
현재 프로젝트	현재 열려 있는 프로젝트 내부의 마크다운 문서만 검색한다.
전체 프로젝트	등록되어 있거나 접근 가능한 모든 프로젝트의 마크다운 문서를 검색한다.

기본 검색 범위는 현재 프로젝트로 설정한다.

⸻

2.2 검색 대상 문서

검색 대상은 마크다운 문서로 제한한다.

지원 확장자 예시는 다음과 같다.

.md
.markdown

마크다운 문서가 아닌 파일은 검색 대상에서 제외한다.

⸻

2.3 검색 방식

문서 검색은 다음 두 가지 방식을 지원한다.

검색 방식	설명
단순 문자열 검색	입력한 문자열이 문서 내용에 포함되어 있는지 검색한다.
Regex 검색	입력한 정규식 패턴과 일치하는 내용을 검색한다.

검색 방식은 사용자가 사이드바에서 선택할 수 있다.

단순 문자열 검색

단순 문자열 검색은 사용자가 입력한 텍스트를 그대로 문서 내용에서 찾는다.

예시:

입력값: inventory system

위 입력값은 문서 내용 중 inventory system 문자열이 포함된 위치를 검색한다.

Regex 검색

Regex 검색은 사용자가 입력한 값을 정규식 패턴으로 해석하여 검색한다.

예시:

TODO|FIXME

위 패턴은 문서 내용 중 TODO 또는 FIXME가 포함된 위치를 검색한다.

정규식 패턴이 올바르지 않은 경우 검색을 수행하지 않고 사용자에게 오류 메시지를 표시한다.

⸻

3. 사이드바 UI 구성

문서 검색 사이드바는 다음 요소로 구성한다.

3.1 검색 입력 영역

* 검색어 입력 필드
* 검색 범위 선택 옵션
    * 현재 프로젝트
    * 전체 프로젝트
* 검색 방식 선택 옵션
    * 단순 문자열
    * Regex
* 검색 실행 버튼

3.2 검색 결과 영역

검색 결과는 문서 단위로 표시한다.

각 검색 결과 항목에는 다음 정보를 표시한다.

* 문서 제목 또는 파일명
* 문서 경로
* 검색어가 포함된 일부 문맥
* 문서 내 매칭 개수

예시:

InventorySystem.md
/project-a/docs/systems/InventorySystem.md
... 아이템 추가 시 inventory system은 ...
3 matches

⸻

4. 검색 결과 클릭 동작

사용자가 검색 결과의 문서를 클릭하면 다음 동작을 수행한다.

1. 우측 문서 뷰어에 해당 마크다운 문서의 전체 내용을 표시한다.
2. 문서 내에서 첫 번째 검색 매칭 위치로 자동 이동한다.
3. 검색된 텍스트를 하이라이트한다.
4. 여러 개의 검색 결과가 있는 경우, 사용자가 이전/다음 검색 위치로 이동할 수 있도록 네비게이션을 제공한다.

⸻

5. 우측 문서 뷰어 기능

우측 문서 뷰어는 선택한 마크다운 문서를 표시하는 영역이다.

5.1 문서 표시

* 선택한 문서의 전체 내용을 렌더링한다.
* 마크다운 문법을 해석하여 제목, 목록, 코드 블록 등을 보기 쉽게 표시한다.
* 검색 결과에서 진입한 경우 검색 매칭 위치를 하이라이트한다.

5.2 검색 위치 네비게이션

문서 뷰어에는 검색된 위치를 이동할 수 있는 네비게이션 기능을 제공한다.

필수 기능은 다음과 같다.

* 현재 매칭 위치 표시
* 전체 매칭 개수 표시
* 이전 매칭 위치로 이동
* 다음 매칭 위치로 이동

예시:

2 / 5
[이전] [다음]

5.3 자동 스크롤

검색 결과 문서를 열면 첫 번째 매칭 위치로 자동 스크롤한다.

사용자가 이전/다음 버튼을 누르면 해당 매칭 위치로 스크롤하고, 현재 선택된 매칭 항목을 강조 표시한다.

⸻

6. 하이라이트 규칙

검색된 텍스트는 문서 뷰어에서 시각적으로 구분되도록 하이라이트한다.

하이라이트는 두 가지 상태를 가진다.

상태	설명
일반 매칭	검색어와 일치한 모든 위치
현재 선택된 매칭	현재 네비게이션으로 선택된 위치

현재 선택된 매칭은 일반 매칭보다 더 명확하게 강조한다.

⸻

7. 예외 처리

7.1 검색어가 비어 있는 경우

검색어가 비어 있으면 검색을 수행하지 않는다.

표시 메시지 예시:

검색어를 입력해 주세요.

7.2 검색 결과가 없는 경우

검색 결과가 없으면 결과 영역에 안내 메시지를 표시한다.

검색 결과가 없습니다.

7.3 Regex 패턴 오류

Regex 검색 모드에서 잘못된 정규식이 입력된 경우 오류 메시지를 표시한다.

올바르지 않은 정규식 패턴입니다.

7.4 문서 로드 실패

검색 결과를 클릭했지만 문서를 불러올 수 없는 경우 우측 뷰어에 오류 메시지를 표시한다.

문서를 불러올 수 없습니다.

⸻

8. 사용자 흐름

8.1 기본 검색 흐름

1. 사용자가 사이드바에서 검색어를 입력한다.
2. 검색 범위를 선택한다.
3. 검색 방식을 선택한다.
4. 검색을 실행한다.
5. 검색 결과 목록이 표시된다.
6. 사용자가 결과 문서를 클릭한다.
7. 우측 뷰어에 문서가 표시된다.
8. 첫 번째 매칭 위치로 자동 이동한다.
9. 사용자는 이전/다음 버튼으로 검색 위치를 이동한다.

⸻

9. 기능 요구사항

* 검색 대상은 마크다운 문서로 제한한다.
* 검색 범위는 현재 프로젝트와 전체 프로젝트를 지원한다.
* 검색 방식은 단순 문자열과 Regex를 지원한다.
* 검색 결과는 문서 단위로 표시한다.
* 각 문서별 매칭 개수를 표시한다.
* 검색 결과 클릭 시 우측 뷰어에 문서를 표시한다.
* 검색된 위치를 하이라이트한다.
* 첫 번째 매칭 위치로 자동 이동한다.
* 이전/다음 검색 위치 네비게이션을 제공한다.
* Regex 오류, 검색 결과 없음, 문서 로드 실패 상태를 처리한다.

⸻

10. 권장 추가 기능

필수 기능은 아니지만 사용성을 높이기 위해 다음 기능을 고려할 수 있다.

* 대소문자 구분 옵션
* 검색 결과 정렬 옵션
* 최근 검색어 저장
* 검색 결과 내 프로젝트명 표시
* 검색 실행 단축키
* 검색어 입력 시 자동 검색
* 현재 문서 내 검색 결과만 보기
