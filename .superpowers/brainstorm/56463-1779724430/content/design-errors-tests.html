설계 — 섹션 4: 에러 처리 & 테스트 전략

마지막 섹션. 실패 케이스 처리와 검증 전략.

에러 처리 경계
⚡ Main Process 에러
  • 파일 없음 → IPC error 반환, Renderer 빈 상태 표시
  • 권한 거부 → 에러 메시지 + 폴더 재선택 유도
  • SQLite 실패 → 로그 기록 후 인메모리 fallback
  • 프로젝트 경로 소실 → 목록에서 경고 배지 표시
⚛️ Renderer 에러
  • Markdown 파싱 실패 → raw 텍스트 fallback 표시
  • Mermaid 문법 오류 → 다이어그램 위치에 오류 메시지
  • 컴포넌트 충돌 → React ErrorBoundary로 격리
  • 이미지 경로 오류 → alt 텍스트로 대체
