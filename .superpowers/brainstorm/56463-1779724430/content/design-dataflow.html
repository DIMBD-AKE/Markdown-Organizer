설계 — 섹션 3: 핵심 데이터 흐름

3가지 핵심 시나리오의 완전한 흐름

① 문서 열기 흐름
사용자 FileNode 클릭
→
fileTreeStore.selectFile(path)
↓
window.api.readFile(path) 호출
preload contextBridge
→
Main: ipcMain.handle('read-file')
↓
fs.readFile() → UTF-8 string 반환
viewerStore.setContent(raw)
→
remark 파이프라인 처리
↓
Shiki 하이라이트 → Mermaid 렌더 → TOC 추출
MarkdownRenderer 표시
+
TocPanel 업데이트
↓
SQLite project_state 업데이트 (last_file, scroll_pos)
② 앱 시작 — 상태 복원 흐름
Main: app.whenReady()
→
SQLite open + schema 확인
↓
app_settings 읽기 → 윈도우 크기/위치 복원
BrowserWindow 생성
→
Renderer mount
↓
App init: window.api.getAppState() 호출
projectStore 복원
→
fileTreeStore 복원
→
마지막 문서 자동 열기
↓
스크롤 위치 복원 → chokidar 감시 시작
③ 파일 변경 감지 → 자동 갱신
chokidar 파일 변경 감지
→
200ms debounce
↓
변경 유형 판별: 추가 / 수정 / 삭제
win.webContents.send('file-changed', payload)
↓
Renderer: useEffect 리스너 수신
현재 열린 파일이면
→
자동 리로드
트리 노드만 변경이면
→
fileTreeStore만 업데이트
